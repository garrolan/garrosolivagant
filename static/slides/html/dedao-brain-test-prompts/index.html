使用方式 · How to test
02 / 17
拍攝時的最短路徑

打開、複製、貼上、觀察。

這份簡報不是要背稿，而是把測試流程放在螢幕上。你只需要翻到某一頁，按「複製」，再貼到得到大腦裡看它怎麼回答。

01
選一頁
每頁是一個測試能力，例如搜尋、金句定位、跨時間回顧。
02
複製提示詞
每張提示詞卡右上角都有按鈕，點一下就會把原句放到剪貼簿。
03
記錄反應
觀察它是精準、發散、卡住，還是意外給出有價值的回答。

Test 01

基礎搜尋能力

驗證小步能否精準搜尋得到 APP 內容，而不是只丟出一串連結。

適用場景
  • 確認小步對得到內容的熟悉度。
  • 測試能否直接回答定義性問題，而非返回列表。
幫我找萬維鋼《精英日課》裡關於刻意練習的內容
吳軍的財商課裡，他怎麼定義財商？
得到上有沒有《納瓦爾寶典》？是電子書還是聽書？

跨時間資料調用

驗證小步能否精確定位到特定日期的筆記或學習記錄。

適用場景
  • 想找回某天散步時的錄音內容。
  • 測試長期記憶的精確度。
我2026年5月21日散步時錄的那條關於GETSEEDAI錄音卡的筆記，核心觀點是什麼？
我今天聽了萬維鋼的《探索與利用》那講，跟我之前記的創作筆記有什麼關聯？
我2026年3月5日那天主要做了什麼？幫我回顧一下

私董會角色扮演

驗證小步能否模擬特定老師的語氣，把學過的內容轉成可用建議。

適用場景

碰撞式提問

驗證小步能否發現不同領域內容之間的關聯，幫你看見自己沒注意到的模式。

適用場景

Test 12

綜合實戰

把多個功能串起來，模擬真實使用場景，也是影片最適合收束或高潮的段落。

適用場景
  • 想看小步在複雜場景下的綜合能力。
  • 適合做為影片高潮段落。
我剛聽完萬維鋼的課，記了幾條筆記，又想起之前關於創作的思考，幫我：1）找出這些內容的關聯 2）拷問一下我的創作策略 3）把結論整理成一條可發的筆記
我這週學了財商課，又記了關於自由的思考，幫我回顧這週的學習，然後看看這兩個主題有什麼碰撞
我剛錄了一段關於YouTube計畫的思考，幫我：潤色一下→點評亮點→拷問漏洞→最後整理成可分享的內容

最後提醒

失敗也是內容。

每頁一個測試，每個測試拍 1-2 分鐘。可以從簡單到複雜：基礎搜尋、金句定位、跨時間調用、私董會、碰撞、拷問、潤色、點評、發芽、GET 日報、時效性、綜合實戰。

Async
發芽功能是異步推送，拍攝時要說明結果會稍後推送到卡片。
Daily Review
GET 日報基於昨天的記錄，拍攝前要確保昨天有足夠內容。
Real Test
如果某個測試沒答好，反而能讓影片更真實，也更有觀察價值。
